Automated PowerPoint Chart Generation Project Summary

Overview

This Python project automates the creation of a PowerPoint slide that displays the active Jira Epics the team is working on, organized by Stakeholders. It retrieves data from Snowflake, organizes it by priority, and presents it as a color-coded heatmap table in PowerPoint. As the work was done at a previous employer I'm not permitted to share the actual code.

Key Steps and Workflow

  • Database Connection: Connects to Snowflake using credentials stored in an RData file.
  • Data Preparation: Fetches active Epics, sorts them by priority, and organizes them by Stakeholder.
  • Data Cleaning: Replaces missing values with 99999, indicating inactive or low-priority Epics.
  • Heatmap Generation: Uses color-coding to visually represent priority levels, with green shades for high priority and grey for inactive data.
  • PowerPoint Slide Creation: Adds a table layout, title, and timestamp to the slide using the python-pptx library.
  • Row Limiting: Ensures a maximum of 8 rows per stakeholder to maintain readability, adding a note if items are hidden.

Purpose of the Project

The goal of this project is to automate reporting and eliminate the manual effort involved in creating a manual PowerPoint dashboards. It saves time, ensures consistency, and provides stakeholders with a clear, up-to-date view of Epic priorities.

Output Example

The generated PowerPoint slide will include:

  • Table headers for each stakeholder
  • Rows of Epics, sorted by priority and color-coded in a heatmap.
  • A note section indicating hidden rows or report limitations.
  • A timestamp at the top right showing when the report was last refreshed.

Conclusion

This project demonstrates the power of Python in automating reporting workflows. It integrates Snowflake for data access and python-pptx for PowerPoint automation, providing a streamlined, professional solution for tracking Jira Epics and their priorities.
